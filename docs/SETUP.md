(Gemini billed to your GCP credits). Project id throughout: **`taskko`**.

## B. Firebase credentials

A Firebase project **is** a GCP project, so the same `taskko` project powers
Auth, Firestore, and Vertex AI.

### B1. Web config — `NEXT_PUBLIC_FIREBASE_*` (admin web UI)
Firebase console → ⚙ **Project settings** → **General** → scroll to **Your apps**.
If there's no **Web** app yet, click **</>** (Add app), name it "Taskko Admin",
register. Copy the `firebaseConfig` values into `admin/.env.local`:

| firebaseConfig field | env var |
|---|---|
| `apiKey` | `NEXT_PUBLIC_FIREBASE_API_KEY` |
| `authDomain` | `NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN` |
| `projectId` | `NEXT_PUBLIC_FIREBASE_PROJECT_ID` (`taskko`) |
| `storageBucket` | `NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET` |
| `messagingSenderId` | `NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID` |
| `appId` | `NEXT_PUBLIC_FIREBASE_APP_ID` |

These are **public** by design (they ship in the browser bundle) — not secrets.

### B2. Service account — `FIREBASE_SERVICE_ACCOUNT` (server, also used by Vertex)
Firebase console → ⚙ **Project settings** → **Service accounts** →
**Generate new private key** → downloads a JSON file. This **is a secret**.

Put it into `admin/.env.local` as a single line. Easiest is base64 (our code
accepts raw JSON or base64):
```powershell
# PowerShell — base64-encode the downloaded file:
[Convert]::ToBase64String([IO.File]::ReadAllBytes("C:\path\to\serviceAccount.json"))
```
Then: `FIREBASE_SERVICE_ACCOUNT=<paste-the-base64>`

### B3. Grant the admin claim
The admin portal + the app's "Try as admin" need a user with the `admin` claim.
1. Sign up once in the app with `admin@taskko.app` (creates the user).
2. From `admin/`, run (using the downloaded key):
   ```powershell
   node scripts/set-admin-claim.mjs admin@taskko.app --key C:\path\to\serviceAccount.json
   ```
3. Sign out / back in so the token refreshes. That account now reaches the admin portal.

## C. Gemini API (on your GCP credit project `taskko-498611`)

Simpler than Vertex: the **Gemini API** (`generativelanguage.googleapis.com`)
uses a plain **API key**. It's enabled on `taskko-498611`, so usage bills to
your $25 credit. No service account for AI.

### C1. Enable billing + the API (once)
1. **console.cloud.google.com** → project **`taskko-498611`** → **Billing** →
   redeem the **$25 voucher** at https://console.cloud.google.com/billing/redeem
   and confirm the project is linked.
2. APIs & Services → Library → **"Gemini API"** (`generativelanguage.googleapis.com`)
   → **Enable** (you already did this).

### C2. Create the API key
APIs & Services → **Credentials** → **Create credentials → API key** → copy it
(an `AIza…` key). Recommended: **Restrict key** → API restrictions → allow only
the **Generative Language API**.

### C3. Env var (`admin/.env.local` / Vercel)
```
GEMINI_API_KEY=<the AIza… key>
# GEMINI_MODEL=gemini-2.0-flash   # optional (default)
```
`lib/gemini.ts` reads `GEMINI_API_KEY`. `FIREBASE_SERVICE_ACCOUNT` (from §B2,
the Firebase account) is separate — Admin-SDK token verification only.

> Cost note: `gemini-2.0-flash` is inexpensive; $25 covers a lot of dev/demo
> traffic. Keep `maxOutputTokens` bounded (already done).
